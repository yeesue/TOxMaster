## 错误记录

### 错误1：C++标准版本兼容性问题
**发生时间**: 2026-03-29  
**任务**: 任务1 - 修复RAT_FUNC除零问题  
**严重程度**: 🔴 高 - 编译失败

#### 错误描述
使用了C++17特性的 `<optional>` 头文件和 `[[deprecated]]` 属性，导致在Qt 5.9.9 / MinGW 5.3.0环境下编译失败。
```
xcp/ratfunc_converter.h:29:20: fatal error: optional: No such file or directory
```

#### 错误原因
1. **环境认知不足**: 未提前确认目标编译环境的C++标准版本支持
2. **盲目使用新特性**: 直接使用了C++17特性，未考虑向后兼容
3. **验证流程缺失**: 编写代码后未先进行编译验证

#### 环境约束
| 环境 | 版本 | C++标准支持 |
|-----|-----|------------|
| Qt | 5.9.9 | C++11为主 |
| MinGW | 5.3.0 | 部分C++14 |
| 配置 | `CONFIG += c++11` | C++11 |

#### 解决方案
1. **替代 `std::optional`**: 创建自定义 `RatFuncResult` 结构体
```cpp
// 不兼容的代码
std::optional<qreal> toPhys(qreal intValue) const;

// 兼容的代码
struct RatFuncResult {
    bool success;
    qreal value;
    bool hasValue() const { return success; }
    qreal getValue() const { return value; }
};
RatFuncResult toPhys(qreal intValue) const;
```

2. **移除 `[[deprecated]]`**: 改用注释说明
```cpp
// 不兼容的代码
[[deprecated("Use safeXxx instead")]]
qreal oldFunction();

// 兼容的代码
// 旧版函数（保留向后兼容，建议使用safe版本）
qreal oldFunction();
```

#### 经验教训
1. **先确认环境再编码**: 开始任务前检查 `.pro` 文件中的 `CONFIG` 设置
2. **使用最低兼容标准**: 代码应满足项目配置的最低标准要求
3. **增量编译验证**: 每完成一个文件修改就尝试编译

#### 后续预防措施
- 所有新代码必须兼容C++11标准
- 避免使用的C++14/17特性：
  - `std::optional` → 自定义结构体或 `QPair<bool, T>`
  - `std::variant` → 联合体或枚举+成员变量
  - `[[deprecated]]` → 注释说明
  - 结构化绑定 `auto [a, b] = ...` → 手动解包
  - `if constexpr` → 模板特化或运行时判断
  - `std::string_view` → `QString` 或 `const char*`
