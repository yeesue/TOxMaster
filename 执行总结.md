## 任务1：修复RAT_FUNC除零问题

### 执行状态：✅ 已完成
### 执行时间：2026-03-29

### 问题分析
原 `RatFuncConvFromIntToPhys()` 和 `RatFuncConvFromPhysToInt()` 函数存在以下问题：
1. 未处理分母为零的情况（导致程序崩溃）
2. 未检查系数列表长度不足的情况
3. 存在浮点精度累积误差
4. 无错误日志记录

### 解决方案

#### 1. 新建文件
| 文件 | 路径 | 说明 |
|-----|-----|-----|
| `ratfunc_converter.h` | `XCP_COMM/xcp/` | 安全转换器类定义 |

| `tst_ratfunc.cpp` | `tests/xcp_comm_tests/` | 单元测试文件 |

#### 2. 关键改进
- **除零保护**：使用epsilon比较（默认1e-10）检查分母是否接近零

- **错误返回**：使用 `std::optional<qreal>` 明确表示转换失败
- **错误日志**：转换失败时记录详细错误
- **缓存优化**：`RatFuncConverterCache` 类避免重复创建转换器

#### 2. 关键特性
```cpp
// 错误码枚举（包含协议定义+自定义）
enum XCPErrorCode {
    ERR_SUCCESS = 0xFF,         // 成功
    ERR_CMD_UNKNOWN = 0x00,     // 未知命令
    ERR_CMD_SYNTAX = 0x01,      // 语法错误
    ERR_CMD_BUSY = 0x02,        // 命令忙
    // ... 协议定义的错误码
    
    // 自定义通信层错误码
    ERR_TIMEOUT = 0xF0,         // 超时
    ERR_NOT_CONNECTED = 0xF1,   // 未连接
    ERR_COMMUNICATION = 0xF2,   // 通信错误
    // ...
};

// 错误信息结构
struct XCPErrorInfo {
    XCPErrorCode code;
    QString name;           // 英文名称
    QString description;    // 中文描述
    QString suggestion;     // 恢复建议
    XCPErrorSeverity severity;  // 错误严重程度
    bool recoverable;       // 是否可恢复
    int retryCount;         // 建议重试次数
};
```

#### 3. 实现内容
- **完整错误码覆盖**：协议定义的14个错误码 + 7个自定义错误码
- **错误严重程度**：None/Info/Warning/Error/Critical 五级
- **恢复建议**：每个错误码都有对应的解决方法
- **可恢复性判断**：`isRecoverable()` 和 `canRetry()` 函数
- **格式化输出**：`formatError()` 函数提供统一的日志格式

#### 4. 使用方式
```cpp
// 原代码
qDebug()<<"CONNECT Error."<<"ErrorCode=0x"<<QString::number(CTO_RESPacket[1], 16);

// 新代码
handleXCPError(CTO_RESPacket[1], "CONNECT");
// 输出: [XCP错误 0x01 CMD_SYNTAX] 命令语法错误：参数格式或值无效 | 建议：检查命令参数是否正确 | 上下文：CONNECT
```

#### 2. 关键特性
```cpp
// 超时配置结构体
struct XCPTimeConfig {
    quint32 connectTimeout = 1000;      // 连接超时
    quint32 commandTimeout = 1000;      // 普通命令超时
    quint32 daqConfigTimeout = 2000;    // DAQ配置超时
    quint32 uploadTimeout = 3000;       // 上传超时
    quint32 downloadTimeout = 3000;     // 下载超时
    quint32 calPageTimeout = 2000;      // 标定页切换超时
    quint32 seedKeyTimeout = 2000;      // Seed/Key操作超时
    quint32 checksumTimeout = 5000;     // 校验和计算超时
    quint32 retryCount = 3;             // 重试次数
    quint32 retryDelay = 100;           // 重试间隔(ms)
};

// 配置管理器（单例模式）
class XCPConfig {
public:
    static XCPConfig& instance();
    
    XCPTimeConfig timeConfig() const;
    void setTimeConfig(const XCPTimeConfig& config);
    
    // 根据命令类型获取对应超时时间
    quint32 getTimeoutForCommand(quint8 commandCode) const;
    
    bool loadFromFile(const QString& path);
    bool saveToFile(const QString& path);
};
```

#### 3. 命令超时映射
| 命令类别 | 命令示例 | 使用超时 |
|---------|---------|---------|
| 连接管理 | CONNECT/DISCONNECT/SYNCH | connectTimeout |
| DAQ配置 | ALLOC_DAQ/SET_DAQ_PTR/WRITE_DAQ | daqConfigTimeout |
| 上传 | UPLOAD/SHORT_UPLOAD | uploadTimeout |
| 下载 | DOWNLOAD/DOWNLOAD_MAX | downloadTimeout |
| 标定页 | SET_CAL_PAGE/GET_CAL_PAGE | calPageTimeout |
| Seed/Key | GET_SEED/UNLOCK | seedKeyTimeout |
| 校验和 | BUILD_CHECKSUM | checksumTimeout |
| 其他 | GET_STATUS/GET_ID等 | commandTimeout |

#### 4. 使用方式
```cpp
// 原代码（硬编码）
if (time.elapsed() > 1000) { ... }

// 新代码（可配置）
quint32 timeout = XCPConfig::instance().getTimeoutForCommand(CTO_CMDPacket[0]);
if (static_cast<quint32>(time.elapsed()) > timeout) { ... }
```

## 任务5：添加A2ML解析器

### 执行状态：✅ 已完成
### 执行时间：2026-03-29

### 问题分析

A2ML是A2ML语法定义语言，用于描述IF_DATA中的数据结构。原代码中：
1. Quex词法分析器配置了`skip_range`跳过了A2ML
2. A2ML节点的parse()函数只是保存原始内容，不解析
3. 缺少A2ML词法和语法分析器

### 解决方案

#### 1. 新建文件

| 文件 | 路径 | 说明 |
|-----|-----|-----|

| `a2ml_parser.cpp` | `A2L_Praser/ASAP2/` | A2ML语法分析器实现 |

#### 2. 关键特性
**A2ML词法分析器（A2mlLexer）**：
- 支持关键字：`struct`, `enum`, `taggedstruct`, `taggedunion`, `block`
- 支持数据类型：`uchar`, `uint`, `ulong`, `char`, `float`, `double`, `sbyte`, `sword`, `slong`, `ubyte`, `uword`
- 支持符号：`{}`, `[]`, `()`, `;`, `=`, `,`, `*`
- 支持字符串、数字、标识符

**A2ML语法分析器（A2mlParser）**：
- AST节点类型：
  - `A2mlStruct` - 结构体定义
  - `A2mlEnum` - 枚举定义
  - `A2mlTaggedStruct` - 带标签结构体
  - `A2mlTaggedUnion` - 带标签联合体
  - `A2mlBlock` - 块定义
  - `A2mlDataType` - 数据类型
  - `A2mlTaggedMember` - 带标签成员

#### 3. A2ML语法结构示例

```
struct Protocol_Layer {
  uint;  /* XCP protocol layer version */
  uchar; /* MAX_CTO */
  taggedstruct {
    ("OPTIONAL_CMD" enum { "GET_ID" = 250 })*;
    "SEED_AND_KEY_EXTERNAL_FUNCTION" char[256];
  };
};

taggedunion Daq_Event {
  "FIXED_EVENT_LIST" taggedstruct { ("EVENT" uint)*; };
  "VARIABLE" taggedstruct { block "AVAILABLE_EVENT_LIST" taggedstruct { ("EVENT" uint)*; }; };
};
```

### 迁移对照表

| 旧代码 | 新代码（推荐） |
|--------|---------------|
| `qDebug() << "msg"` | `LOG_DEBUG() << "msg"` |
| `qInfo() << "msg"` | `LOG_INFO() << "msg"` |
| `qWarning() << "msg"` | `LOG_WARN() << "msg"` |
| `qCritical() << "msg"` | `LOG_ERROR() << "msg"` |

### 兼容性说明

1. **完全兼容**：继续使用`qDebug()`等，没有自定义通过Qt的消息处理器转发到Logger
2. **推荐迁移**：新代码使用`LOG_XXX()`宏，可获得更精确的文件名、行号信息
3. **LogWin自动连接**：创建LogWin时自动连接到Logger

### 使用示例

```cpp

    LOG_PERF_START("database_query");
    // ... 执行数据库查询 ...

    // 自动作用域性能统计

    // ... 执行耗时操作 ...
}

// 方式2：保留原qDebug写法仍然有效

void oldFunction() {
    qDebug() << "This still works";
    // 自动转发到Logger
```
