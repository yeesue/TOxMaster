## 二、第一阶段：内存安全（P0-紧急）

### 2.1 问题分析

**风险点**：
1. 跨线程信号槽传递裸指针，接收者需负责delete，易造成内存泄漏
2. 线程异常终止时动态分配的缓冲区无法释放
3. 未使用智能指针管理动态对象

### 2.2 解决方案

**创建智能指针工具类** (`XCP_COMM/common/smart_ptr.h`)：

```cpp
#ifndef SMART_PTR_H
#define SMART_PTR_H

#include <QSharedPointer>

// 线程安全的字节数组智能指针
using ByteArrayPtr = QSharedPointer<quint8>;

inline ByteArrayPtr makeByteArray(int size) {
    return ByteArrayPtr(new quint8[size], [](quint8* p) { delete[] p; });
}

// 线程安全的字符数组智能指针
using CharArrayPtr = QSharedPointer<char>;

inline CharArrayPtr makeCharArray(int size) {
    return CharArrayPtr(new char[size], [](char* p) { delete[] p; });
}

#endif // SMART_PTR_H
```

**修改信号槽定义**：
```cpp
// 修改前
signals:
    void RESDataReady(quint8 *data, quint32 len);

// 修改后
signals:
    void RESDataReady(ByteArrayPtr data, quint32 len);
```

## 三、第二阶段：线程优化（P0-紧急）

### 3.1 问题分析

**问题点**：
1. `usleep(10)` 导致CPU占用过高
2. 线程退出仅靠bool标志，缺乏同步机制
3. 线程生命周期管理不规范

### 3.2 解决方案

**引入条件变量机制**：

```cpp
class XCP_R_Thread : public QThread
{
private:
    QMutex m_mutex;
    QWaitCondition m_condition;
    bool m_running = true;
};

void XCP_R_Thread::run()
{
    while (m_running)
    {
        // 使用SDK的超时机制替代忙等待
        status_R_XNET_Stream = nxReadFrame(session_R_XNET_Stream, 
                                           frameBuffer_R_Stream, 
                                           numOfBytesForFrames_Stream,
                                           10,  // 10ms超时
                                           numOfBytesReturned_Stream);
        
        if (!m_running) break;
        
        if (status_R_XNET_Stream == 0 && numOfBytesReturned_Stream > 0) {
            // 处理数据...
        }
    }
}

void XCP_R_Thread::stop()
{
    QMutexLocker locker(&m_mutex);
    m_running = false;
    m_condition.wakeAll();
}
```

### 3.3 验收标准

- [x] 编译无错误
- [ ] CPU占用率下降（目标：<5%空闲时）
- [ ] 线程启动/停止响应及时（<100ms）
- [ ] 无死锁或竞态条件

1. 样式分散在多个.cpp文件中

    background-color: #00aa00;
    border-radius: 4px;
}

LED[label="off"] {
    background-color: #aa0000;
    border-radius: 4px;

    gridline-color: #d0d0d0;

## 五、第四阶段：信号槽升级（P1-重要）

### 5.1 问题分析

**问题点**：
1. 使用旧式SIGNAL/SLOT宏，编译时不检查
2. 运行时才发现连接错误
3. 性能略低于新式连接

### 5.2 解决方案

**修改示例**：

```cpp
// 修改前
connect(xcpPollThread, SIGNAL(measPamsValueUpdated()), 
        this, SLOT(fromReadSMToMeasVars()));

// 修改后
connect(xcpPollThread, &XCP_Polling_Thread::measPamsValueUpdated,
        this, &Xcp_Fun_Thread::fromReadSMToMeasVars);
```

**处理重载信号**：
```cpp
// 使用QOverload选择重载版本
connect(comboBox, QOverload<int>::of(&QComboBox::currentIndexChanged),
        this, &MyClass::onIndexChanged);

// 或使用static_cast
connect(this, static_cast<void(PARAM::*)(qreal)>(&PARAM::valueUpdated), ...);
```

## 六、第五阶段：参数显示优化（P2-一般）

### 6.1 问题分析

**问题点**：
1. QStandardItemModel大数据量性能差
2. 无虚拟滚动
3. 刷新机制可优化

### 6.2 解决方案

**创建自定义TableModel**：

```cpp
class ParamTableModel : public QAbstractTableModel
{
public:
    enum Columns { ColName = 0, ColValue, ColUnit, ColCount };
    
    int rowCount(const QModelIndex &parent = QModelIndex()) const override;
    int columnCount(const QModelIndex &parent = QModelIndex()) const override;
    QVariant data(const QModelIndex &index, int role = Qt::DisplayRole) const override;
    
    void updateAllValues();  // 批量更新，只发射一次dataChanged信号
    
private:
    QList<PARAM*> m_params;
    QVector<double> m_cachedValues;  // 缓存值
};
```
