## 三、潜在问题分析
### 3.1 崩溃风险

#### 3.1.1 内存管理问题

| 问题 | 位置 | 风险等级 | 说明 |
|------|------|----------|------|
| **原始指针管理** | `node.h/cpp`, `item.h/cpp` | 🔴 高 | Node类使用 `char* name` 和 `QList<Node*>`，析构用 `qDeleteAll`，存在内存泄漏风险 |
| **全局变量初始化** | `globals.h/cpp` | 🟡 中 | 静态成员变量初始化顺序不确定，可能导致未定义行为 |
| **DLL接口内存** | `a2l_parser_interface.cpp` | 🔴 高 | `QLibrary` 动态加载，返回指针未做生命周期管理 |
| **线程资源泄漏** | `xcp_thread.cpp` | 🟡 中 | 线程未正确退出可能导致资源泄漏 |

**问题代码示例**：
```cpp
// globals.h - 静态成员初始化顺序问题
class Globals {
public:
    static QList<A2L_VarChar*> charPamList_global;  // 初始化顺序不确定
    static QList<A2L_VarMeas*> measPamList_global;
};

// a2l_parser_interface.cpp - 内存泄漏
QMap<QString, QString>* GetMeasNodeInfoByName(QString fullFileName, QString reqMeasName) {
    // 返回 new 出来的指针，调用者需负责释放，容易泄漏
    return new QMap<QString, QString>();  // 🔴 谁来释放？
}

// xcpmaster.cpp - 原始指针
void XCPMaster::getMeasValueByName(...) {
    A2L_VarMeas* meas = ...;  // 🔴 原始指针，无所有权
}
```

#### 3.1.2 线程安全问题

| 问题 | 位置 | 风险等级 | 说明 |
|------|------|----------|------|
| **全局变量竞态** | `Globals::*` | 🔴 高 | 多线程访问全局列表未加锁 |
| **信号槽跨线程** | `XCP_Thread` → `XCPMaster` | 🟡 中 | Qt信号跨线程需要 QueuedConnection |
| **DAQ数据竞争** | `DAQ_Processor` | 🟡 中 | DAQ数据处理和UI更新可能竞争 |

**问题代码示例**：
```cpp
// xcp_fun_thread.cpp - 竞态条件
void Xcp_Fun_Thread::run() {
    while (running) {
        // 直接访问全局变量，无锁保护
        for (auto meas : Globals::measPamList_global) {  // 🔴 竞态
            // ...
        }
    }
}

// UI更新线程安全问题
void XCPMaster::updateMeasValue() {
    // 跨线程更新UI，可能崩溃
    plotWin->updatePlot();  // 🔴 非UI线程调用
}
```

#### 3.1.3 空指针解引用

| 问题 | 位置 | 风险等级 | 说明 |
|------|------|----------|------|
| **DLL加载失败** | `a2l_parser_interface.cpp` | 🔴 高 | `QLibrary::resolve` 返回 nullptr 后直接调用 |
| **节点查找失败** | `a2l_project.cpp` | 🟡 中 | `child()` 返回 nullptr 后未检查 |
| **驱动初始化失败** | `xcp_thread.cpp` | 🟡 中 | 硬件初始化失败后继续操作 |

**问题代码示例**：
```cpp
// a2l_parser_interface.cpp - 空指针
void A2L_PARSER_INTERFACE::init_A2LParser() {
    QLibrary dllLib(a2lParserDllPath);
    if (dllLib.load()) {
        Init init = (Init)dllLib.resolve("Init");
        if (init) {
            init();  // OK
        } else {
            qDebug() << "resolve error";
            // 🔴 没有返回，继续执行可能崩溃
        }
    }
}

// a2l_project.cpp - 空指针
QString A2L_Project::getMeasValueByName(QString name) {
    A2L_VarMeas* meas = getMeasPtrByName(name);
    return meas->getValueString();  // 🔴 meas 可能是 nullptr
}
```

### 3.2 运行效率问题

#### 3.2.1 字符串处理
| 问题 | 位置 | 影响 | 说明 |
|------|------|------|------|
| **QString频繁拷贝** | 全项目 | 🟡 中 | 函数参数传递使用值传递而非 const& |

```cpp
// 全项目 - QString值传递

QString result;

```

```cpp
}

```
